# Qwen Agent Lab V2.2 · 大方向

V2.2 已落地为“工具生态与流程干预”版本，核心方向是让 Agent 从少量内置工具扩展为可治理、可确认、可审计的工具系统。

## 已落地方向

- 工具治理元数据：`category / dangerous / requiresConfirmation / timeoutMs / maxResultChars`。
- Human-in-the-loop：`memory_write` 等需要确认的工具在执行前挂起，等待用户允许或拒绝。
- 工具生态扩展：`file_search`、`data_analyzer`、`chart_generator`、`report_builder`、`web_fetch`、`memory_search`、`memory_write`。
- 安全边界：单工具超时、结果截断、SSRF 防护、报告 HTML 转义、本地记忆容量上限。
- 验收状态：见 `docs/v2.2-acceptance.md`，当前自动化测试为 97/97 通过。

## 后续候选

- `document_parser`：需要评估 PDF / DOCX / XLSX 解析依赖。
- `browser_reader`：需要 Playwright 或浏览器运行时，建议放到 V2.3 或独立实验。
