文档日期：2026-06-04

本文件记录 V2.0「真·Agent」的验收项、方法与结果。自动化项可通过 `npm test` 复跑；接口/场景项通过本地启动后实测。

## 一、验收环境

- 启动：`npm start`（端口 5173），应用入口 `http://127.0.0.1:5173/app`
- 自动化测试：`npm test`（`node --test tests/*.test.js`）
- 关键配置：`AGENT_MAX_ITERATIONS=5`、`AGENT_MAX_TOOL_STEPS=8`、`BOCHA_API_KEY`（留空=Mock）

## 四、接口实测示例

```bash
# 非流式 Agent：真实执行 calculator
curl -s -X POST http://127.0.0.1:5173/api/qwen/chat \
  -H "Content-Type: application/json" \
  -d '{"mode":"agent","input":"用计算器算 (15+27)*3","model":"qwen-plus"}' \
  | jq '.data.toolSteps'

# Chat：同一任务，不应产生工具调用
curl -s -X POST http://127.0.0.1:5173/api/qwen/chat \
  -H "Content-Type: application/json" \
  -d '{"mode":"chat","input":"用计算器算 (15+27)*3","model":"qwen-plus"}' \
  | jq '.data.toolSteps'   # => []
```

## 五、遗留 / 后续

- **真实联网搜索**：默认 `WEB_SEARCH_PROVIDER=qwen`，复用现有 `QWEN_API_KEY` 通过千问内置搜索（`enable_search`）返回实时结果，无需额外凭据；已实测可返回训练时点之后的事实。可选切换为 `bocha`（需博查 key，返回结构化网页链接）或 `mock`。
- HTTP 层（`/api/qwen/chat`）的 Agent 行为依赖真实模型，未纳入离线单测；逻辑通过执行器单测 + 本地实测覆盖。
