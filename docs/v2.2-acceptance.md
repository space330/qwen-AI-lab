## 四、安全与漏洞排查 (Security Review)

*   **SSRF (服务端请求伪造) 防御与深度分析**：
    - `web_fetch` 中的 [guardFetchUrl](file:///d:/Qwen%20Agent%20Lab/server/tools.js#L829) 构筑了黑名单安全防线，拦截了 `10.*`、`192.168.*`、`172.16-31.*` 等内网私有网段以及 AWS 元数据服务网段 `169.254.169.254`。
    - *深度审计提示*：由本地第一（local-first）的桌面端定位所致，系统设计上允许了 `localhost`、`127.0.0.1` 和 `::1` 以便 Agent 能够调试及抓取本地运行的服务。但需提防在公网多租户部署时通过 DNS 重绑定（DNS Rebinding）绕过 Hostname 过滤器的风险。
*   **XSS (跨站脚本注入) 防御**：
    - `report_builder` 中组装 HTML 时，对所有章节的 `heading` 和 `body` 全部通过 `esc()` 进行 HTML 实体编码，无法从后端输出逃逸恶意的标签段落。
    - 前端在接收这些 HTML 时仍然会通过 `sanitizeAnswerHtml` 深度 DOM Walk，剔除脚本与事件（如 `onload`），达到双重保险。
*   **数据解析漏洞 (Data Coercion Bug) 排查与修复**：
    - *发现漏洞*：排查发现 `data_analyzer` 中的 `aggregate`、`top_n` 和 `filter` 动作在处理 CSV 中空单元格或非法字符串（如 `N/A`、空字符串 `""`）时直接调用 `Number(row[col])`，这在 JS 中会把空字符串强制转换为 `0` 并误作为有效数值加入计算。特别在平均数 (`avg`) 计算中，这会导致分母偏大，平均数偏小。
    - *修复逻辑*：我们重构了 `server/tools.js`，引入了前置 `String(rawVal).trim() !== ""` 判空验证，对于非法或空字符串统一转换为 `NaN`，使其不参与 `Number.isFinite` 校验。新增的单元测试已闭环验证此修复。
*   **内存与死锁防御**：
    - 所有的挂起 resolver 均被 `ownedKeys` 并发集所管理。在 `try-finally` 及 `req.on("close")` 场景下，均会主动调用 `releaseOwned("disconnected")` 清理挂起事件，杜绝了由于客户端提前断开造成的后台协程永久挂起与内存泄露。
