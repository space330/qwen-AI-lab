# Qwen Agent Lab v2.0.0 发布说明

发布日期：2026-06-04

V2.0 把 Agent 模式从「占位」升级为**真正的工具调用循环（ReAct）**：后端按模式分流，Agent 可自主调用内置工具、把结果回填模型并继续推理，全过程通过流式协议实时呈现为「工具链时光轴」。

## Agent 执行架构

```
用户输入(agent) → buildAgentMessages(系统提示 + 工具说明 + 上下文 + 文件)
  → runAgent 循环：
      callQwen(带 tools)
        ├─ 返回 tool_calls → 执行工具 → 结果作为 role:"tool" 消息回填 → 再次 callQwen
        └─ 返回最终答案(无 tool_calls) → 收束
  → formatSuccess(content + 完整 toolSteps) → 输出
```

- 控制循环：`server/agentExecutor.js` 的 `runAgent`（async generator），与传输/服务商解耦，便于单测。
- 路由分流：`server/index.js` 的 `/api/qwen/chat` 与 `/api/qwen/chat/stream` 均按 `mode` 区分；Agent 走 `executeAgent`，Chat 走单次模型调用。
- 流式与非流式**共用同一套执行逻辑**（`executeAgent`），保证两种接口行为一致。

## 内置工具

| 工具 | 说明 | 备注 |
|------|------|------|
| `calculator` | 精确数学计算 | 自实现解析器（非 eval），支持 `+ - * / % ^ ()` 与小数，除零报错 |
| `file_reader` | 按行读取当前会话上传的附件 | 让 Agent 按需读取片段，而非一次性塞入整文件 |
| `web_search` | 联网检索实时信息 | 默认 `provider=qwen`：复用现有 DashScope key 走千问内置搜索（`enable_search`），零新增凭据；可选 `bocha`（结构化网页结果，需 key）或 `mock` |

## Agent 与 Chat 的区别

| 维度 | Chat | Agent |
|------|------|-------|
| 工具调用 | 不调用任何工具 | 按需自动调用内置工具 |
| 后端链路 | 单次模型调用 | 多步 ReAct 工具循环 |
| 输出 | 简短结构化 HTML、不含图表占位 | 工具链时光轴 + 最终结构化 HTML |
| 流式事件 | `status` / `result` / `error` | 额外 `tool_start` / `tool_result` / `tool_error` |

## 工具调用循环与限制

- `AGENT_MAX_ITERATIONS`（默认 5）：模型「请求工具」的轮次上限。
- `AGENT_MAX_TOOL_STEPS`（默认 8）：**实际执行的工具总步骤**上限（单轮可能返回多个工具调用）。
- 达到任一上限后，执行器会**强制要求模型停用工具并给出最终答案**；最终响应通过 `agentStepLimitHit` 与 `agentTruncatedReason`（`iteration_limit` / `tool_step_limit`）标记。

## 统一的 toolSteps 数据契约

流式事件、最终响应、前端状态、IndexedDB 持久化使用同一结构：

```json
{
  "step": 1,
  "tool": "calculator",
  "status": "running | completed | error",
  "args": { "expression": "19 * 23" },
  "result": "{\"result\":437}",
  "error": null,
  "durationMs": 12
}
```

- 最终响应 `data.toolSteps` 包含完整的状态、结果、错误与耗时。
- 非流式客户端也能拿到完整工具执行历史。

## 搜索服务配置

```env
# qwen（默认，复用 DashScope key 的千问内置搜索，无需额外凭据）| bocha | mock
WEB_SEARCH_PROVIDER=qwen
QWEN_SEARCH_MODEL=qwen-plus        # provider=qwen 时用于检索的模型（需支持 enable_search）
BOCHA_API_KEY=                     # provider=bocha 时使用，留空回退 Mock，仅后端读取
BOCHA_BASE_URL=https://api.bochaai.com/v1
BOCHA_TIMEOUT_MS=15000
```

- **provider=qwen（默认）**：复用现有 `QWEN_API_KEY`，通过千问 `enable_search` 返回实时结果；开箱即用，无需任何新账号。
- **provider=bocha**：返回结构化网页链接，需博查 key；缺 key 回退 Mock（前端显示 **Mock** 徽标）。
- **provider=mock**：强制本地占位（离线/演示）。
- 真实检索失败时**不静默降级为 Mock**，返回真实失败原因，Agent 可基于已有信息继续作答。

## 工具错误恢复机制

- 工具抛错 → 推送 `tool_error`，并把错误作为工具结果回填，模型可换策略或继续。
- 未知工具 → 记为 `tool_error`，不会使整个请求崩溃。
- 工具步骤的参数/结果/错误在前端渲染时全部经 HTML 转义。

## 已知限制

- Agent 建议使用支持函数调用的模型（如 `qwen3.7-max` / `qwen-max` / `qwen-plus`）；不支持的模型会优雅降级为普通回答。
- `web_search` 的博查返回字段以官方文档为准，已做容错抽取；真实检索质量取决于服务方。
- 多步工具循环在慢模型下累计耗时可能较长，已有心跳与等待提示。
